인스턴스 종료 시 미과금이란 종량제 인스턴스의 **종료 시 미과금 옵션을 선택**하여, **종료** 상태가 된 인스턴스(CPU, 메모리)의 요금을 더는 부과하지 않는 것을 가리킵니다. [클라우드 디스크](https://intl.cloud.tencent.com/document/product/213/2255)(시스템 디스크와 데이터 디스크), 공용 네트워크 대역폭(대역폭 과금) 및 이미지 등의 모듈은 계속 과금됩니다.

## 사용 제한

**인스턴스 종료 시 미과금** 기능은 **시스템 디스크와 데이터 디스크 모두 클라우드**인 **종량제 인스턴스**에만 적용할 수 있습니다.
인스턴스 종료 시 미과금 기능은 아래의 시나리오에서는 **적용할 수 없습니다**.
- 인스턴스 시작/종료에는 사용할 수 없습니다.
- 로컬 디스크가 탑재된 인스턴스에는 적용할 수 없습니다.
- 스팟 인스턴스에는 적용할 수 없습니다.
- 연체로 인해 종료된 인스턴스에는 적용할 수 없습니다. 인스턴스 및 연결 리소스가 연체로 종료되면 과금도 함께 정지되며, 컴퓨팅 리소스와 공인 IP도 릴리스됩니다. 계정에 잔액을 충전한 후에는, 모든 리소스가 계속 과금됩니다.

일괄 종료 작업 도중, 종료 시 미과금을 지원하는 인스턴스와 지원하지 않는 인스턴스가 동시에 포함된 경우에는,
- 종료 시 미과금을 지원하는 인스턴스는 종료 후, **CPU 메모리 과금이 정지됩니다.**
- 종료 시 미과금을 지원하지 않는 인스턴스는 종료 후, **정상 과금을 유지합니다.**

## 시작 영향

인스턴스 종료 시 미과금 기능을 활성화하면, 인스턴스 종료 후 **다음과 같은 영향이 있습니다**.
1. 인스턴스의 CPU와 메모리가 **보관되지 않으므로** 인스턴스를 다시 시작할 때 **실패할 수 있습니다.** 이때 재시작하거나, 다른 시간에 재시작을 시도할 수 있습니다.
2. 공인 IP 주소가 할당된 인스턴스를 종료할 경우, 공인 IP 주소가 **자동으로 릴리스됩니다.** 인스턴스 시작 후, 인스턴스는 새로운 공인 IP 주소가 할당됩니다. 개인 IP는 영향을 받지 않으며, 변경 없이 보관됩니다.
 **공인 IP의 변경을 원하지 않을 경우, 공인 IP를 EIP로 전환한 후 인스턴스를 종료하시기 바랍니다.** 인스턴스에 바인딩된 EIP는 종료 후에도 유지되며, 재시작 후에도 유지됩니다. 종료 후 EIP 유휴 기간의 비용은 과금되지 않습니다.
3. 종료 기간에는 **인스턴스 시작 외의** 다른 작업을 지원하지 않습니다(설정 변경, 디스크 변경, 네트워크 변경, 시스템 재설치, 인스턴스 재시작, 비밀번호 재설정, 이름 변경 관련 작업 등). **작업이 필요할 경우 시작 후 진행하시기 바랍니다.**
4. 설정 변경, 디스크 변경, 시스템 재설치 시의 기타 유지보수 작업을 실행할 경우 인스턴스를 종료해야 하며, 종료 기간에는 종료 시 미과금 정책을 **실행하지 않습니다.**
