자체구축 원본 서버 및 라이브 방송 원본 콘텐츠가 있고 Tencent Cloud를 통해 라이브 방송을 재생해야 하는 경우, CSS 재생 도메인에 원본 서버 정보를 설정하여 라이브 방송 콘텐츠의 원본을 가져올 수 있습니다. 설정이 완료되면 CSS를 통해 원본 풀 스트림 및 라이브 방송 콘텐츠 배포가 가능합니다. 본 문서에서는 원본 서버 정보 설정 방법을 소개합니다.

## 주의 사항
- 원본 서버 설정 기능은 관련 정보 설정 완료 1일 후 적용됩니다. 
- 원본 서버 설정 기능 활성화한 후, 해당 재생 도메인은 StreamName과 다른 푸시 도메인 매칭을 통한 풀 스트림을 지원하지 않으며, 해당 도메인에서는 워터마크, 트랜스 코딩, 녹화, 화면 캡처, 음란물 감지 등의 기능을 사용할 수 없습니다. 


## 전제 조건
- [CSS 콘솔](https://console.cloud.tencent.com/live)에 로그인되어 있어야 합니다.
- 라이브 방송 원본 서버가 구축되어 있어야 합니다.
- **재생 도메인**이 추가되어 있어야 합니다.

## 작업 절차
1. [**도메인 관리**](https://console.cloud.tencent.com/live/domainmanage)로 이동하여 설정할 **재생 도메인** 또는 오른쪽에 있는 **관리**를 클릭해 도메인 상세 페이지로 이동합니다.
2. **고급 설정**을 선택하여 **원본 서버 설정**태그를 조회합니다. 
3. 원본 서버 설정 모듈의 **편집**을 클릭하여 다음과 같이 원본 서버 정보를 설정할 수 있습니다.
 1. **원본 서버 설정**버튼을 움직여 활성화합니다. 
 2. **Origin-pull 프로토콜**을 선택하고 **재생 프로토콜**을 클릭합니다. 
 3. **호스트 원본 주소**(IP 주소 또는 도메인 입력 지원)를 입력합니다.
 4. **저장**을 클릭하면 설정이 저장됩니다. 

>?**Origin-pull 프로토콜**이란 CSS에서 원본 서버에 요청한 소스를 풀 스트림 시 지원하는 형식입니다.** 재생 프로토콜**이란 라이브 방송 콘텐츠를 라이브 방송 CDN을 통해 배포할 때의 재생 프로토콜입니다.
>- Origin-pull 프로토콜이 FLV/RTMP인 경우, 재생 프로토콜은 FLV, RTMP, HLS를 지원합니다. (**FLV 재생 프로토콜을 선택하면 RTMP를 직접 재생할 수 있습니다**).
>- Origin-pull 프로토콜이 HLS인 경우, 재생 프로토콜은 HLS만 지원합니다.
>- Origin-pull 프로토콜은 중복 선택할 수 없으며, 재생 프로토콜은 중복 선택할 수 있습니다.
