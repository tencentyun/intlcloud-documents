### CSS에는 어떤 요금제가 있나요? 어떤 비용을 지불해야 하나요?
CSS의 요금제에는 기본적으로 기본 요금과 부가 요금이 있습니다. 이 중 기본 요금은 트래픽/대역폭을 기준으로 과금됩니다. 즉, 가속 원본 서버와의 연결로 인해 발생하는 다운스트림 비용으로, 라이브 방송 콘텐츠를 시청하는 사용자가 있으면 트래픽/대역폭 비용이 발생한다고 생각하면 됩니다. 트래픽 과금 방식과 대역폭 과금 방식 중 1가지를 선택할 수 있으며, 과금 방식은 [CSS 콘솔](https://console.cloud.tencent.com/live)을 통해 전환할 수 있습니다. 자세한 단가는 [트래픽 대역폭 요금]을 참조하십시오.
부가 요금에는 트랜스 코딩, 화면 캡처, 성인물 감지 기능이 포함되며, 해당 4개 기능은 기본적으로 꺼져 있습니다. 해당 기능을 활성화하고 사용한 후에는 이에 상응하는 비용이 발생하며, 자세한 단가는 해당 [기능 요금]을 참조해 주십시오.

### 구매한 CSS 트래픽 패키지를 소진한 후에는 서비스가 정지되나요?
CSS 트래픽 패키지를 소진한 후에는 과금 방식이 후불 일별 트래픽 과금 방식으로 기본 전환되며 서비스가 정지되지 않습니다. Tencent Cloud 계정에 잔액이 0원이상 남아있으면 트래픽 패키지를 소진한 후에도 계속해서 CSS 서비스를 이용할 수 있습니다.

### 라이브 방송 업스트림 푸시 스트리밍은 유료인가요?
CSS 업스트림 푸시 스트리밍은 무료이며 다운스트림의 트래픽만 유료입니다.

### 라이브 방송 트랜스 코딩의 과금 방식은 어떻게 되나요? 트랜스 코딩 예상 요금은 어떻게 산정하나요?
CSS 트랜스 코딩은 실제 트랜스 코딩의 코딩 방식, 해상도, 해당 시간에 따라 과금되며, 라이브 방송 혼합 스트리밍과 워터마크 추가 또한 트랜스 코딩 모듈 프로세스를 거쳐 트랜스 코딩 비용이 발생합니다. 자세한 사항은 [트랜스 코딩 요금]을 참조하십시오.
다중 사용자가 시청하는 상황에서 동일한 라이브 방송 스트리밍, 동일한 부호율에 대해서는 트랜스 코딩 1회 비용만 과금됩니다.

### 라이브 방송 트랜스 코딩을 사용하지 않았는데 왜 트랜스 코딩 청구서가 발행된 건가요?
CSS 트랜스 코딩에는 라이브 방송 실시간 트랜스 코딩, 클라우드 단의 혼합 스트리밍 및 워터마크 추가 3가지 종류가 있으며, 혼합 스트리밍 또는 워터마크 기능을 사용한 경우 트랜스 코딩 비용이 청구됩니다.

### 연체 여부는 어떻게 알 수 있나요?
[Tencent CSS 콘솔](https://console.cloud.tencent.com/live)에 로그인한 후, 오른쪽 상단의 [Billing Center]을 클릭하면 요금 개요 페이지로 들어가며 사용 가능한 잔액이 0원 미만인 경우 연체 상태가 됩니다. CSS 및 기타 서비스에 영향이 미치지 않도록 즉시 충전해 주십시오.
