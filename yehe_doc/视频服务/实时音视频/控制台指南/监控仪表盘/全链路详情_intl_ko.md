전체 링크는 발신측에서 수신측으로 멀티미디어 데이터를 보낼 때 발생하는 모든 연결 링크 프로세스를 말하며, 멀티미디어 통화의 원활성을 보장하기 위해서는 양호한 네트워크 + 디바이스 안전성이 필요합니다. 따라서 개발자는 사용자 통화의 전체 링크를 점검할 때 두 가지 사항을 가장 먼저 점검해야 합니다.

## 작업 순서

1. TRTC 콘솔에 로그인하여 [모니터링 대시보드](https://console.cloud.tencent.com/trtc/monitor)에서 점검할 방을 선택합니다.
2. 방 ID 또는 오른쪽에 있는 [통화 상세 정보 보기]를 클릭하여 [통화 상세 정보](https://intl.cloud.tencent.com/document/product/647/39070) 페이지로 이동합니다.
3. 수신/발신측 데이터 표시 열에서 점검할 사용자 ID를 선택하고 다음 방법으로 전체 링크 상세 정보 페이지에 들어가 두 사용자 간의 전체 링크 상세 정보를 확인합니다.
	- [수신측 시점]에서 오른쪽에 있는 [발신측 선택하여 상세 정보 보기]에서 발신측 사용자 ID를 선택합니다.
	- [발신측 시점]에서 오른쪽에 있는 [상세 정보 보기]를 클릭합니다.

![](https://main.qcloudimg.com/raw/dbd80dcb851766e32acf8020e5f2555d.png)




## 데이터 분석

전체 링크 상세 정보 페이지 표시되는 데이터는 [비디오], [오디오], [화면 공유] 세 가지 유형으로 나뉘며, 발신측 및 수신측별 링크 데이터를 확인할 수 있습니다.
![](https://main.qcloudimg.com/raw/8f1a21141b61590e6b7a13a705030900.png)
전체 링크 상세 정보에서는 [네트워크 상태 분석](#internet), [디바이스 상태 확인](#equipment) 두 가지 사항에 대해 분석을 진행합니다.

[](id:internet)
### 네트워크 상태 분석
이상적인 상황에서 네트워크 전송 시 패킷 손실 및 딜레이, 많은 대역폭이 발생하지 않습니다. 그러나 실제 상황에서는 크고 작은 패킷 손실, 전송 지연, 불안정 현상이 종종 발생하며 네트워크 대역폭 또한 제한이 존재해 네트워크 상태 분석 시 다음 사항을 중점적으로 확인해야 합니다.

[](id:packet_loss)
#### 네트워크 패킷 손실
데이터 그래프의에서 네트워크에 패킷 손실이 발생한 시간에 빨간색 선이 표시됩니다.

| 패킷 손실률                         | 네트워크 상태 설명             |
| ------------------------------ | ------------------------ |
| = 0                            | 최적                     |
| < 2%                           | 기본적으로 양호                 |
| &gt; 5%                        | 양호하지 않음                     |
| &gt; 10%(또는 지속적으로 패킷 손실 발생) | 현재 네트워크에 비교적 심각한 정체 발생 |

[](id:bit_rate)
#### 비트 레이트
정상적인 상황에서 비디오 및 오디오 비트 레이트의 상하 파동 범위는 ±10% 이내입니다. **현재 비트 레이트가 갑자기 낮아지거나 30% 이상의 파동이 발생하는 경우 현재 네트워크에 정체 또는 지터가 발생했다는 의미입니다.**
> ! 공유 화면의 GOP가 비교적 길어(5초~10초) 정상적인 상황에서는 안정적인 주기의 피크 곡선이 나타나지만, 키 프레임이 있는 경우 해당하는 파고점이 나타납니다.

- 화면 공유 업스트림 비트 레이트 및 네트워크 패킷 손실
  ![](https://main.qcloudimg.com/raw/04dbc42ef897b2d9e462109723d4bd83.png)
- 안정적인 주기의 화면 공유 비트 레이트 피크 곡선
  ![](https://main.qcloudimg.com/raw/536e174c3bc1acf6405fa9b03cb26c19.png)

[](id:frame_rate)
#### 프레임 레이트
정상적인 상황에서 비디오 프레임 레이트는 통상적으로 ≥15 프레임(화면 공유 프레임 레이트는 일반적으로 5~10프레임)이며 안정적으로 유지됩니다. **프레임 레이트에 5프레임 이상의 파동이 발생하거나 프레임 레이트가 10프레임 이하로 떨어지고 지속적으로 회복되지 않는 경우, 일반적으로 현재 네트워크에 정체 또는 지터가 발생했다는 의미로** 사용자가 주관적으로 랙이 발생했다고 인식할 수 있습니다. 프레임 레이트가 너무 낮은 시간에는 데이터 그래프에 빨간색 선이 표시됩니다.

- 비디오 업스트림 프레임 레이트: 수집과 발신 프레임 레이트로 나뉩니다.
  ![](https://main.qcloudimg.com/raw/aece9e6339df38b2d043fc070c798543.png)
- 비디오 렌더링 프레임 레이트: 프레임 레이트가 낮아진 경우 빨간색 선으로 표시되며 랙 시간 길이가 나타납니다.
  ![](https://main.qcloudimg.com/raw/9d61cf98732bcd972542f2f46ba36aee.png)


[](id:equipment)
### 디바이스 상태 확인

디바이스의 안정적이고 정상적인 작동은 멀티미디어 통화를 보장하는 기본 사항으로, 디바이스 상태가 우수한 통화는 시스템 리소스 점유율이 낮고 디바이스 사용을 점유하지 않으며 데이터 수집에 방해가 없습니다. 디바이스 상태 확인 시 다음 정보를 우선적으로 확인합니다.

[](id:cpu)
#### CPU 점유율
CPU 점유율은 시스템의 전체적인 CPU 점유율과 App CPU 점유율을 나타내며, 정상적인 상황에서 시스템 전체 CPU 점유율은 50% 미만으로 낮을수록 양호한 상태입니다. **시스템 전체 CPU 점유율이 85%를 초과하는 경우 프로그램이 응답하지 않거나 응답이 느린 현상이 발생하며, 이 때 빨간색 선이 표시됩니다.**
![](https://main.qcloudimg.com/raw/3685883ac4a44d78481359f17648485d.png)

[](id:voice)
#### 볼륨 크기
- 오디오 **수집 볼륨**은 발신측의 마이크에서 수집한 데이터의 볼륨 크기로, **수집 볼륨 크기 값에 파동이 있으면 마이크가 현재 정상적으로 음성을 수집하고 있다는 의미, 즉 디바이스가 정상적으로 작동되고 있다는 의미입니다.**
- 오디오 **재생 볼륨**은 수신측에서 렌더링 데이터를 디코딩한 후 스피커로 발송하는 볼륨 크기로, **재생 볼륨 크기 값에 파동이 있으면 SDK가 이미 음성을 스피커로 발송했다는 의미, 즉 SDK가 정상적으로 작동되고 있다는 의미입니다.**

정상적인 볼륨 크기는 일반적으로 40dB~80dB 사이로, 40dB 이하인 경우 볼륨이 비교적 작다는 의미입니다. 사용자에게 음성이 들리지 않는 경우 휴대폰 스피커가 음소거되어 있지 않은지, 하드웨어에 고장이 발생하지 않았는지 확인이 필요합니다.
![](https://main.qcloudimg.com/raw/5052ae41e76a81580c4dddf67433a2c2.png)

[](id:resolution)
#### 해상도
비디오 및 화면 공유 해상도는 부가 정보로, 주로 릴레이 라이브 방송 및 녹화 파일 재생을 판단하는 데 사용됩니다. 비디오 해상도가 변경되는 경우 [Content Delivery Network](https://intl.cloud.tencent.com/product/cdn)를 통해 릴레이 라이브 방송 또는 비디오 재생 화면을 보는 시청자, 특히 Web 사용자에게 화면 랙 등 재생기 호환 문제가 발생할 수 있습니다.
![](https://main.qcloudimg.com/raw/422e0c787216954deb3b9c5deea97bdd.png)


>? [해상도](#resolution), [비트 레이트](#bit_rate), [프레임 레이트](#frame_rate) 사이에는 일정의 관계가 존재합니다. 일반적으로 해상도 고정 시 비트 레이트가 높을수록 화면이 더욱 선명해지며, 비트 레이트 고정 시 해상도가 높을수록 화면이 흐려집니다. 해상도, 비트 레이트, 프레임 레이트 매개변수를 적합하게 조정하면 더 좋은 화질의 비디오 화면을 제공할 수 있습니다.

#### 클라이언트 이벤트 확인
클라이언트 이벤트는 App에서 SDK를 호출하는 방법으로 작동되며, 일반적으로 소프트웨어 문제 추적 보조 및 bug 분석에 사용합니다. 사용자가 사용하는 작업 순서를 통해 해당하는 시나리오를 재현하며, 중점적으로 확인할 사항은 다음과 같습니다.

- 입장, 퇴장 이벤트
- 카메라 또는 마이크 켜기, 끄기
- 디바이스 변화: 카메라 전환, 이어폰 연결/해제, 블루투스 이어폰 연결 등
- 푸시 스트리밍 또는 재생 시작, 정지
- 음소거/음소거 취소 작업, 화명 정지/화면 정지 취소 작업
- 네트워크 전환: 예를 들어 4G에서 WIFI로 전환

[이벤트 상세 보기]를 클릭하여 상세 이벤트 리스트 페이지에 들어가 클라이언트 상의 주요 이벤트 작업을 확인할 수 있습니다.
