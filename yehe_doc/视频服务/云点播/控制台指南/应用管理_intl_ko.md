## 작업 시나리오
애플리케이션 관리를 통해 사용자는 해당 애플리케이션의 리소스를 비활성화하거나 폐기할 수 있어 애플리케이션을 더욱 유연하게 관리할 수 있습니다. VOD 애플리케이션의 상태는 정상, 비활성화, 폐기의 세 종류로 나뉘며, 애플리케이션의 라이프사이클과 관련 상태의 의미는 다음 이미지와 같습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/5d453fecd052b987f8219a2fe4e331a2.png)



| 상태 | 의미 |
|---------|---------|
| 정상 | 해당 상태에서 애플리케이션이 표준 사용 상태인 경우 사용자는 구성을 변경하고 관련 비디오 처리 및 미디어 자원 관리 등의 작업을 수행할 수 있습니다. | 
|비활성화됨|해당 상태에서는 사용자의 애플리케이션이 비활성화되고, 해당 애플리케이션 리소스와 구성 파일은 모두 보관되며, 상응하는 과금 항목에 따라 해당 리소스에 대한 요금이 청구됩니다. 그러나 해당 상태에서의 애플리케이션은 구성을 변경할 수 없으며 공용 네트워크에서 액세스할 수 없습니다. |
|폐기됨|해당 상태에서는 애플리케이션 내의 리소스가 완전히 폐기되고, 모든 리소스와 구성 파일이 모두 제거되어 있으며, 리소스와 구성을 복구할 수 없습니다. 서비스 정지 시나리오에 적용됩니다. |

>!
>- 폐기 후 애플리케이션을 다시 활성화할 수 있으며, 활성화한 애플리케이션의 ID(appid/subappid)는 변경되지 않습니다.
>- 데이터 통계에는 약간의 딜레이가 있습니다. 따라서 애플리케이션 폐기 후 데이터 통계에 일정 수준의 딜레이가 발생하는 것은 정상적인 현상으로 사용자의 요금 정산에 영향을 주지 않습니다.
>- 애플리케이션의 상태 변경에는 5~10분이 소요되며, 사용자는 이를 자주 변경하지 마십시오.

애플리케이션은 리소스 관리에 대한 VOD 사용자의 권한과 관련되어 있습니다. 따라서 애플리케이션의 비활성화 및 폐기는 사용자의 신원을 인증한 뒤에만 허용할 수 있는 강제적이고 민감한 권한에 속하는 작업입니다. 신원 인증 후 30분 이내에는 2차 인증이 필요하지 않습니다.
애플리케이션 관리 기능의 작업 페이지는 사용자가 서브 애플리케이션을 활성화했는지 여부에 따라 달라지며, 두 가지 시나리오에 대한 콘솔 가이드는 다음과 같습니다.

## 계정에 서브 애플리케이션이 없는 경우

계정에 서브 애플리케이션이 없는 경우 애플리케이션 관리는 [자주 쓰는 툴]에 위치하며, 사용자는 계정에 직접 서브 애플리케이션을 추가하고 세부 정보를 볼 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/6f80829605ee04ce07b6414f2fc2c405.png)

이 때 사용자는 즉시 세부 정보를 생성하고 볼 수 있습니다.

## 계정에 여러 개의 서브 애플리케이션이 있는 경우

계정에 여러 개의 서브 애플리케이션이 있는 경우 애플리케이션 관리는 관리자 >[애플리케이션 관리]에 위치하며, 사용자가 모든 VOD 애플리케이션의 상태를 변경할 수 있습니다. 사용자는 서브 애플리케이션 상태에서 현재 애플리케이션의 상태를 확인할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/279d7be223412490b95bb0d73e2ed55d.png)

사용자는 ON/OFF 키를 통해 서브 애플리케이션을 활성화/비활성화할 수 있습니다.

## 특이사항:
1. 사용자가 서브 애플리케이션을 보유하지 않은 경우.
	- 메인 애플리케이션을 폐기할 경우, VOD 계정의 리소스와 구성이 모두 삭제됩니다.
	- 메인 애플리케이션을 폐기했다가 다시 활성화하면 VOD 계정도 다시 활성화됩니다. 계정이 연체되어 있는 경우, 잔액이 0인지 확인해 주십시오. 그렇지 않을 경우, 사용할 수 없습니다.

2. 사용자가 서브 애플리케이션을 보유한 경우.
	- 메인 애플리케이션을 폐기하려면 우선 모든 메인 애플리케이션/서브 애플리케이션을 비활성화해야 합니다. 비활성화 상태에서만 폐기 작업을 수행할 수 있습니다.
	- 메인 애플리케이션 폐기 후, 특정 서브 애플리케이션을 다시 활성화하려는 경우, 반드시 먼저 메인 애플리케이션을 활성화해야만 다른 서브 애플리케이션을 다시 활성화할 수 있습니다. 메인 애플리케이션이 폐기된 상태에서는 서브 애플리케이션 사용이 불가합니다.
