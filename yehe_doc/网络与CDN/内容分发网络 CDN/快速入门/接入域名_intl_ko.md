서비스 활성화 후 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인해 메뉴에서 [도메인 관리]를 선택하여 가속 도메인을 추가하면 Tencent Cloud에서 제공하는 가속 서비스를 이용할 수 있습니다. 도메인 추가 후에는 관련 도메인 설정이 전체 네트워크 CDN 가속 노드로 전달되며, 현재 네트워크 서비스에 직접적인 영향을 미치지 않습니다. 정식으로 가속을 활성화해야 하는 경우 CNAME 구성이 필요하며, 자세한 절차는 [CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121)을 참고하십시오.


## 도메인 추가
도메인 관리 페이지에서 [도메인 이름 추가]를 클릭합니다.
![](https://main.qcloudimg.com/raw/020628a1fba4d8529bf3b5be7f459170.png)

도메인 이름 추가 페이지는 세 부분으로 구성되어 있습니다.

- 도메인 속성 설정
- 원본 서버 설정
- 가속 서비스 설정

Tencent Cloud CDN은 선택한 도메인 속성 설정과 원본 서버 설정에 따라 캐시 특성 설정을 추천하며, 수정하지 않고 바로 제출할 수 있습니다.

### Part1. 도메인 속성 설정
도메인 설정 모듈에 비즈니스 서비스 도메인을 입력하고 해당 프로젝트, 가속 지역, 서비스 유형 등을 선택합니다.

### Part2. 원본 서버 설정

비즈니스 원본 서버 관련 정보를 설정하며, 캐시에 리소스가 없을 때 CDN 노드가 원본 서버에서 불러와 캐싱합니다.
![](https://main.qcloudimg.com/raw/947c24efc846a11b25dca0f8ea552422.png)

**설정 항목 상세 설명:**

| 설정 항목   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 원본 서버 유형 | 외부 원본: <br>안정적으로 운영 중인 서비스 서버(원본 서버)를 보유한 시나리오입니다.<br/><br><a href = "https://intl.cloud.tencent.com/product/cos">COS 원본</a>：<br>리소스가 Tencent Cloud COS에 저장되어 있어 바로 bucket을 원본 서버로 지정할 수 있습니다.<br><br/>3rd party COS: <br>Tencent Cloud 이외의 3rd party COS인 경우로, 현재 AWS S3와 Alibaba Cloud만 지원됩니다.<br/>*참고: **일부 플랫폼은 현재 3rd party COS를 지원하지 않습니다. 추후에 지원될 예정입니다.|
| 원본 서버 주소 | 외부 원본: <br/>1. 다수 IP를 원본 서버로 설정 가능하며, Origin-pull은 폴링 Origin-pull을 진행합니다.<br/>2. 포트(0 - 65535)와 가중치(1 - 100) 추가 설정 지원: ‘원본 서버: 포트: 가중치’(포트 생략 가능: ‘원본 서버::가중치’), HTTPS 프로토콜은 현재 443 포트만 지원됩니다. <br/>3. 도메인을 원본 서버로 설정할 수 있습니다. 해당 도메인은 CDN 가속 도메인과 같을 수 없습니다.<br/>**참고: **원본 서버 주소가 이미 연결된 CDN 가속 도메인인 경우, 반복적으로 리졸브할 수 있어 Origin-pull을 정상적으로 진행할 수 없습니다. <br/><br/>COS 원본: <br/>1. Tencent Cloud COS에서 버킷 하나를 원본 서버로 선택합니다.<br/>2. 버킷의 설정과 실제 비즈니스 시나리오에 맞춰 기본 도메인 혹은 정적 웹사이트 유형을 선택합니다. <br/>3. 개인 버킷인 경우 CDN에 권한을 부여하고 Origin-pull 인증을 활성화하여 개인 버킷 액세스를 활성화하십시오.<br/><br/>3rd party COS: <br/>1. 리소스가 3rd party COS에 저장되어 있는 경우, 원본 서버로 유효한 버킷 액세스 주소를 입력하십시오. 현재 지원하는 3rd party: AWS S3및 Alibaba Cloud OSS.<br/>2. 3rd party 개인 버킷에 Origin-pull을 실행하는 경우 유효한 키를 입력하고 Origin-pull 인증을 활성화하여 개인 버킷 액세스를 활성화해야 합니다. 
| Origin-pull 프로토콜 | 원본 서버 지원 상황에 따라 원본 요청 프로토콜 선택:<br/>HTTP: <br/>HTTP/HTTPS 액세스는 모두 HTTP Origin-pull을 사용합니다.<br/><br/>HTTPS:<br/>HTTP / HTTPS 액세스는 모두 HTTPS Origin-pull(원본 서버가 HTTPS 액세스를 지원해야 함)을 사용합니다.<br/><br/>프로토콜 따르기: <br/>HTTP 액세스는 HTTP Origin-pull을 사용하고, HTTPS 액세스는 HTTPS Origin-pull(원본 서버가 HTTPS 액세스를 지원해야 함)을 사용합니다. |
| Host 헤더 | Origin-pull 도메인. CDN 노드에서 Origin-pull을 실행할 때, 액세스한 원본 서버 IP 주소의 구체적인 사이트 도메인입니다. <br/><br/>외부 원본: <br/>현재 가속 도메인. 와일드카드 서브도메인을 연결하면 와일드카드 서브도메인으로 기본 설정되며 실제 Host 헤더는 액세스 도메인이 됩니다. 실제 비즈니스 상황에 맞게 수정할 수 있습니다. <br/><br/>COS 원본:<br/>버킷 액세스 주소로 기본 설정되며, 원본 서버와 동일하고 수정할 수 없습니다. <br/><br/>3rd party COS: <br/>버킷 액세스 주소로 기본 설정되며, 원본 서버와 동일하고 수정할 수 없습니다. |

| 캐시 규정 | 노드 캐시 만료 시간을 설정합니다. 기본적인 파일 캐시 만료 시간은 정적 가속 유형의 경우 모두 원본 서버를 따르며, 다운로드, 스트림 미디어 VOD 가속 유형의 경우 모두 30일입니다. 노드 캐시 만료 시간을 최대로 설정하는 경우 노드 보관 리소스에 영향을 주며 실제 캐시 시간은 상황에 따라 다릅니다. |

## 도메인 소유권 인증[](id:m1)
와일드카드 서브도메인으로 연결했거나 다른 사용자가 연결한 도메인의 경우 또는 신규 도메인을 처음 연결하는 경우, 도메인 소유권 인증이 필요하며, 인증을 통과해야만 도메인 연결 또는 회수가 가능합니다.
도메인 소유권 인증은 DNS 인증 방식이며 인증 절차는 다음과 같습니다.
1. [인증 방법]을 클릭하여 DNS 인증 시 추가해야 하는 리졸브 기록 정보를 불러옵니다. 인증이 완료되기 전까지 페이지를 닫지 마십시오.
   ![img](https://main.qcloudimg.com/raw/c05c0d869a71dbe68092ccee632c2d1a.png)
2. 도메인 리졸브 서비스 제공 업체에서 TXT 유형의 DNS 기록을 추가하고, CPM 기록에는 _cdnauth를, 기록 값에는 인증 방법에서 랜덤 생성된 값을 각각 입력합니다. [DNS 리졸브 DNSPod](https://console.cloud.tencent.com/cns)를 예로 들면 다음과 같습니다.
