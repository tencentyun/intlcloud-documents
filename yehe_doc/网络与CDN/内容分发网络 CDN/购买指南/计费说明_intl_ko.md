#### 계산 방식

전일 CDN 중국대륙 지역의 대역폭 피크치가 X이고 중국 외 각 리전에서 소모가 발생하지 않은 경우, 차등 계산 방식은 다음과 같습니다.

1.  X < 500Mbps인 경우, 청구서 요금은 X*0.0815입니다. 
2. 500Mbps ≤ X <5000Mbps인 경우, 청구서 요금은 X*0.0800입니다. 
3. 5000Mbps ≤ X <50000Mbps인 경우, 청구서 요금은 X*0.0754입니다. 
4.   X ≥ 50000Mbps인 경우, 오프라인 계약으로 더 많은 혜택을 제공해드립니다.

#### 계산 방식

트래픽 과금은 대역폭 과금과 달리 월간 트래픽에 따라 차등 누적되며, 트래픽 일 결산 계산 방식은 다음과 같습니다.

- 1월1일 중국대륙 리전에서 트래픽 소모가 3TB 발생했고 중국 외 리전에서 소모가 발생하지 않은 경우는 다음과 같습니다. 회색으로 표시된 부분은 실제 과금 차등 구간이며 녹색은 1월1일에 발생한 트래픽으로 2TB는 0TB - 2TB 과금 구간에 포함되고 1TB는 2TB - 10TB 과금구간에 포함됩니다. 따라서 1월1일 하루 동안 실제 발생한 비용은 다음과 같습니다. 2 * 1000 * 0.0323 + 1 * 1000 * 0.0308.
  ![img](https://mc.qcloudimg.com/static/img/bfdae242f6cca57421a65e46a96b0c67/image.png)
- 1월2일 중국대륙 리전에서 트래픽 소모가 3TB 발생했고 중국 외 리전에서 소모가 발생하지 않은 경우는 다음과 같습니다. 트래픽 과금 방식은 월간 누적 방식이기 때문에 1월2일에 발생한 트래픽 모두 2TB - 10TB 과금 구간에 포함됩니다. 따라서 1월2일 하루 동안 실제 발생한 비용은 다음과 같습니다. 3 * 1000 * 0.0308.
  ![img](https://mc.qcloudimg.com/static/img/f62d1056c1c2cab249cec62ad6e74ddc/image.png)
- 1월3일 중국대륙 리전에서 트래픽 소모가 7TB 발생했고 중국 외 리전에서 소모가 발생하지 않은 경우는 다음과 같습니다.이 경우 3번의 7TB 중 4TB는 2TB - 10TB 과금 구간에 포함되고 3TB는 10TB - 50TB 과금 구간에 포함됩니다. 따라서 1월3일 하루 동안 실제 발생한 비용은 다음과 같습니다.4 * 1000 * 0.0308 + 3 * 1000 * 0.0277. 
  ![img](https://mc.qcloudimg.com/static/img/954e2d483e31afd411f9a91ebd7f66c8/image.png)

이와 같은 방법으로 월별 일일 비용을 계산하고 2월1일부터 과금이 시작되면 모든 것이 다시 0부터 차등 누적됩니다.이 외에도 [가격 계산기](https://buy.cloud.tencent.com/calculator/cdn)를 이용해 예상 비용을 계산해 볼 수 있습니다.



### 일 피크 대역폭으로 월 평균을 계산하여 과금

1. 1월1일부터 정식으로 과금이 시작되었다고 가정할 경우 계약 가격은 다음과 같습니다.  P USD/Mbps/월.
2. 유효일: 발생한 소모가 > 0 이라면 유효일로 계산됩니다. 
3. 1월에 총 14일의 유효일이 있다고 가정할 경우, 이 14일 동안 일별 통계포인트 288개의 최대값은 다음과 같습니다.  Max_1、Max_2、Max_3... Max_14，과금 대역폭 Average(Max_1, Max_2, ..., Max_14)，1월 비용：Average(Max_1, Max_2, ..., Max_14) * P * 14 / 31.



### 월간 대역폭 95% 과금

1. 1월1일부터 정식으로 과금을 시작한 경우 계약 가격은 다음과 같습니다. P USD/Mbps/월.
2. 유효일: 발생한 소모가 > 0 이라면 유효일로 계산됩니다. 
3. 1월에 총 14일의 유효일이 있다고 가정할 경우, 이 14일 동안의 모든 통계 포인트인 14 * 288개에서 상위 5%를 제외한 나머지 중 가장 높은 값인 Max95가 과금 대역폭 기준이 됩니다. 따라서 1월 요금은 다음과 같습니다.  Max95 * P * 14 / 31.

## 과금 방식 선택

>!
>
> 사용 중에 선택한 과금 방식이 실제 비즈니스 상태에 적합하지 않은 경우 과금 방식을 변경할 수 있습니다. 자세한 내용은 [과금 방식 변경](https://intl.cloud.tencent.com/document/product/228/32326)을 참조하십시오.

**방식 선택:**
CDN 은 두 종류의 과금 방식을 제공합니다. 트래픽 과금과 대역폭 중 자신의 서비스 상황에 맞춰 적합한 과금 방식을 선택할 수 있습니다. 

**계산 예시:**
전날 00:00 ~ 23:59에 발생한 중국대륙 지역의 트래픽이 200GB이고, 대역폭 피크치가 40Mbps인 경우 그래프는 다음과 같습니다.
![img](https://mc.qcloudimg.com/static/img/3ecfe86a031782ebeaf0b1f7595cc69f/image.png)

트래픽 과금 방식의 경우 지불할 비용은 다음과 같습니다. 200 * 0.037 = 7.4 USD

대역폭 과금 방식의 경우 지불할 비용은 다음과 같습니다. 40 * 0.094 = 3.76 USD 

대역폭 과금 방식이 더 저렴하므로, 이 경우에는 대역폭 과금 방식 선택을 권장합니다.
