## 작업 시나리오
Tencent Cloud의 네트워크는 [기본 네트워크와 VPC 사설 네트워크](https://intl.cloud.tencent.com/document/product/215/31807)로 구분되며 사용자에게 각기 다른 품질의 서비스를 제공하고 있습니다. 더욱 편리하고 효율적인 네트워크 관리를 위해 이러한 서비스를 기반으로 아래와 같은 서비스를 제공합니다.
- **네트워크 변경**
  - 기본 네트워크를 사설 네트워크로 변경: 단일 CDB의 기본 네트워크를 사설 네트워크(VPC)로 변경할 수 있습니다.
  - 사설 네트워크A를 사설 네트워크B로 변경: 단일 CDB의 사설 네트워크A를 사설 네트워크B로 변경할 수 있습니다.
- **사용자 정의 IP 설정**

## 주의 사항
- 네트워크를 변경하면 해당 인스턴스의 개인 IP가 바뀌고 24시간 후에 기존의 액세스 IP를 사용할 수 없도록 기본 설정되어 있으므로, 클라이언트 프로그램을 제때 수정하시기 바랍니다. 기존 IP주소의 회수 시간을 0으로 설정하면 네트워크 변경 후 기존 IP 주소를 즉시 회수합니다.
- 기본 네트워크를 사설 네트워크로 변경한 후에는 다시 되돌릴 수 없으며, CDB를 사설 네트워크로 변경한 후에는 다른 사설 네트워크 및 기본 네트워크의 클라우드 서비스와 통신할 수 없습니다.
- 변경한 CDB가 마스터 인스턴스이고, 읽기 전용 인스턴스 혹은 재해 복구 인스턴스가 마운트되어 있다면, 마스터 인스턴스의 네트워크 변경 시 마운트된 읽기 전용 인스턴스 혹은 재해 복구 인스턴스의 네트워크는 마스터 인스턴스에 따라 자동으로 변경되지 않으니 수동으로 변경해야 합니다.

## 작업 순서
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 뒤, 인스턴스 리스트에서 인스턴스 이름 혹은 '작업' 열의 [관리]를 클릭하여 인스턴스 상세 페이지에 접속합니다.
2. 인스턴스 기본 정보의 '소속 네트워크'에서 네트워크 간 변경 유형에 따라 [VPC 네트워크로 변경] 혹은 [네트워크 변경]을 클릭합니다.
3. 팝업된 대화 상자에서 VPC 및 대응하는 서브넷을 선택한 뒤 [확인]을 클릭합니다.
>?
>- IP 주소를 지정하지 않았다면 시스템에서 자동으로 할당합니다.
>- 타깃 VPC는 MySQL 인스턴스가 위치한 리전의 VPC 네트워크만 선택할 수 있습니다.
>- CVM이 있는 VPC를 선택하시길 권장합니다. 그러지 않으면, CVM에서는 내부 네트워크를 통해 MySQL(2개의 VPC 사이에 [피어링 연결](https://intl.cloud.tencent.com/document/product/553/18827)혹은 [CCN](https://intl.cloud.tencent.com/document/product/1003/30049)을 생성한 경우는 제외)에 액세스할 수 없습니다.
