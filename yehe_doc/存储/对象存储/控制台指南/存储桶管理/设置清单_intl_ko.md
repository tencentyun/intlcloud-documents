## 소개

[COS 콘솔](https://console.cloud.tencent.com/cos5)에서 버킷의 리스트 기능을 활성화할 수 있습니다. 리스트 기능을 활성화하면 객체 속성, 설정 상세 정보 등에 관한 리스트 보고서를 매일 또는 매주 귀하의 버킷에서 확인할 수 있습니다. 리스트에 대한 세부 사항은 [리스트 기능 개요](https://intl.cloud.tencent.com/document/product/436/30622)를 참고하십시오. 다음은 버킷 리스트 기능 활성화 방법입니다. 예약 리스트 및 인스턴트 리스트를 필요에 따라 선택할 수 있습니다.

>!
> - 사용자는 버킷별로 여러 개의 리스트 작업을 설정할 수 있습니다.
> - 리스트 작업 실행 중에는 직접 객체 콘텐츠를 읽을 수 없으며, 객체 메타데이터 등의 속성 정보의 스캔 작업만 할 수 있습니다.
> 

## 전제 조건

생성된 버킷에 대한 자세한 내용은 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참고하십시오.

## 작업 단계

### 예약 리스트 추가

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인합니다.
2. 왼쪽 메뉴에서 [버킷 리스트]를 클릭합니다.
3. 리스트 기능을 활성화할 버킷(원본 버킷)을 클릭해 이동합니다.
4. 왼쪽 메뉴에서 [기본 설정]>[리스트 설정] 선택 후 [리스트 추가]를 클릭합니다.

5. **리스트 추가**페이지에서 다음과 같은 설정이 가능합니다.
 1. 기본 정보

    - **규칙 상태**: 리스트 규칙 활성화 여부입니다. 활성화/비활성화 선택 가능합니다.
    - **리스트 이름**: 출력 리스트 이름을 입력합니다.
    - **타깃 버킷**: 리스트를 저장하는 버킷입니다. 기본 설정된 타깃 버킷은 원본 버킷이며, 타깃 버킷은 반드시 원본 버킷과 같은 리전에 있어야 합니다.
    - **리스트 보고서 접두사(선택 사항)**: 타깃 버킷용 접두사를 입력하면 공용 위치의 매니페스트 파일을 그룹 나누기하여 기본값으로 초기화할 수 있습니다.
 2. 필터링 조건

    -**파일 범위**: 리스트 객체의 범위 선택입니다. 전체 버킷 또는 지정 파일 접두사 선택 가능합니다. 미선택 시 전체 버킷으로 설정됩니다.
	- **객체 버전**: 리스트에 객체 버전의 포함 범위를 선택합니다(모든 버전 포함/현재 버전만 포함). 선택하지 않을 경우 기본적으로 현재 버전만 포함합니다.
	- **필터링 태그(선택 사항)**: 필터링을 통해 태그가 동일한 객체를 리스트로 이동합니다. 미입력 시 필터링되지 않습니다.
	- **시간 필터링(선택 사항)**: 필터링을 통해 수정 시간이 일정 시간 이후 또는 일정 기간 이내로 설정된 객체를 리스트에 넣습니다. 설정하지 않을 경우 기본적으로 필터링되지 않습니다.
	- **리스트 정보**: 객체 크기, 스토리지 유형, ETag, 버킷 복사 상태, 멀티파트 업로드 상태, 최신 업데이트 날짜, 태그와 같은 리스트 보고서에 포함해야 하는 객체의 정보를 선택할 수 있습니다. 선택하지 않을 경우 기본값은 전체 선택입니다.
>! ETag(엔터티 태그)는 객체의 해시입니다. ETag는 객체의 내용 변경만 반영할 뿐, 객체의 메타데이터 변경은 반영하지 않습니다. ETag가 반드시 객체 데이터의 MD5 요약은 아니며, 이는 객체의 생성 방식과 암호화 방식에 따라 결정됩니다.
>
 3. 출력 포맷

	- **출력 포맷**: 기본적으로 CSV 포맷으로 출력됩니다.
	- **생성 주기**: 리스트를 내보낼 주기를 선택합니다(매일/매주). 기본적으로 매일 내보냅니다. 예를 들어, 당일 오후 3시에 리스트가 추가되면 늦어도 보통 다음 날 오전 6시까지는 매니페스트 파일이 생성되어 타깃 버킷에 전송됩니다.
	- **리스트 암호화**: 선택한 리스트의 서버 암호화 여부를 선택할 수 있습니다.
		 - 비암호화: 리스트를 암호화 하지 않을 경우의 기본 항목입니다.
		 - SSE-COS: COS 호스팅 키의 서버 암호화를 사용하여 보고서를 암호화합니다. 더 많은 정보는 객체 스토리지 개발자 가이드의 [SSE-COS 암호화](https://intl.cloud.tencent.com/document/product/436/18145)를 참고하십시오.
	- **액세스 권한**: 기본값은 비활성화 입니다. 액세스 권한을 활성화한 후 다음 단계를 진행할 수 있습니다.
 4. 정보 확인

버킷의 리스트 설정 정보를 확인합니다. 수정이 필요한 경우 [이전]을 클릭합니다.
6. 설정 정보에 오류가 없음을 확인한 후, [확인]을 클릭합니다. COS는 앞으로 매일 또는 매주 자동으로 매니페스트 파일을 생성하여 귀하가 설정한 타깃 버킷으로 전송합니다.
>? 리스트 기능이 생성한 리스트 보고서 포맷과 내용의 세부 사항은 [리스트 기능 개요](https://intl.cloud.tencent.com/document/product/436/30622)를 참고하십시오.
>

### 인스턴트 리스트 생성

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인합니다.
2. 왼쪽 메뉴에서 [버킷 리스트]를 클릭한 후 리스트 기능을 활성화하고 싶은 버킷(원본 버킷)을 클릭합니다.
3. 왼쪽 [기본 설정]>[리스트 설정] 클릭 후 리스트 규칙을 선택합니다. [인스턴스 리스트 생성]을 클릭하면 인스턴스 리스트가 생성됩니다.

>? 인스턴트 리스트를 생성하려면 기존 리스트의 작업열에서 [인스턴트 리스트 생성]을 클릭합니다.
> 만약 리스트를 설정하지 않았다면, 리스트를 먼저 설정하여 인스턴트 리스트를 생성하십시오.
