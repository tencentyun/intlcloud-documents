## 소프트웨어 로그인

COSBrowser 데스크톱은 영구 키 로그인, Tencent Cloud 계정 로그인 및 공유 링크 로그인의 3가지 로그인 방법을 지원합니다. 동일 계정 다중 디바이스 멀티 포인트 로그인을 지원합니다.

### 영구 키 로그인

Tencent Cloud API 키(즉 SecretID, SecretKey)를 통해서 로그인할 수 있으며, 해당 키는 액세스 관리 콘솔의 [API 키 관리](https://console.cloud.tencent.com/cam/capi) 페이지에서 생성 및 획득할 수 있습니다. 로그인 완료 후 키는 **세션 내역**에 저장되어 다음에도 계속해서 사용할 수 있습니다. 소프트웨어 로그인 인터페이스는 다음과 같습니다. 다른 설정 항목 관련 설명은 다음과 같습니다.
-**버킷/액세스 경로**: 입력 후 해당 경로로 빠르게 이동할 수 있습니다. 현재 로그인에 사용한 키에 버킷 또는 버킷 하위 일정 디렉터리의 권한만 있는 경우 필수 기입 항목입니다. 형식: Bucket 또는 Bucket/Object-prefix. 예: examplebucket-1250000000 또는 examplebucket-1250000000/folderName.
-**비고**: 현재 입력된 영구 키를 설명할 수 있습니다. 예: 운영자, 목적 등. 키 인터페이스 이력에서 세션 내역을 관리할 때 서로 다른 SecretID를 구별할 수 있습니다.
- **세션 기억**:
    - 선택하지 않으면 이번 로그인만 유효하며, 로그아웃 시 입력된 Tencent Cloud API 키가 지워집니다(현재 키가 세션 내역에 저장되어 있으면 세션 내역에서 제거됨).
    - 선택하면 입력된 Tencent Cloud API 키가 기억되어 세션 내역에서 관리할 수 있습니다.

>! COSBrowser는 프로젝트 키를 통한 로그인을 지원하지 않습니다.
>
<img src="https://main.qcloudimg.com/raw/5fcb01823ae8220e035e86bfd12f047c.png" width="90%">



### Tencent Cloud 계정 로그인

[Tencent Cloud 계정 로그인]을 클릭하고 팝업 창에서 Tencent Cloud 계정으로 COSBrowser 데스크톱에 로그인합니다. Tencent Cloud 계정에서 지원하는 로그인 방법에는 WeChat, 이메일, QQ, WeChat 공식 계정, WeCom 및 서브 계정 로그인이 있습니다.



### 공유 링크 로그인

[[공유폴더](#share)]는 공유 링크와 액세스 코드를 생성할 수 있는 기능으로, **링크 공유**와 **액세스 코드**를 통해 COSBrowser 데스크톱에 임시로 로그인할 수 있습니다.

#### 2. 버킷 상세 정보 조회

버킷 리스트의 오른쪽에 있는 [자세히] 아이콘을 클릭하여 버킷의 상세 정보를 조회할 수 있으며, 버킷 상세 정보에는 버킷 이름, 리전, 액세스 권한, 버전 제어 상태가 포함되어 있습니다.



<span id="count"></span>

#### 3. 통계 데이터 조회

버킷 리스트의 오른쪽에 있는 [...]>[통계]를 클릭하여 버킷의 통계 데이터를 조회할 수 있으며, 버킷 데이터에는 스토리지 용량 및 객체 수가 포함되어 있습니다.

<span id="acl"></span>

#### 4. 권한 관리

COSBrowser는 버킷 및 파일의 권한 관리를 지원합니다.

- 버킷 권한 관리: 버킷 리스트의 오른쪽 작업 열에서 [권한 관리]를 클릭하여 작업할 수 있습니다.
- 객체 권한 관리: 파일의 오른쪽 작업 열에서 [권한 관리]를 클릭하여 작업할 수 있습니다.

>? COS 권한에 대한 자세한 설명은 [ACL 개요](https://intl.cloud.tencent.com/document/product/436/30583)를 참고하십시오.
>

#### 5. 버전 제어 설정

COSBrowser는 버킷 버전 제어 활성화/비활성화를 지원합니다.

버킷 리스트의 오른쪽 작업 열에서 [버전 제어]를 클릭하여 작업할 수 있습니다.

>?버전 제어에 대한 자세한 설명은 [버전 제어 개요](https://intl.cloud.tencent.com/document/product/436/19883)를 참고하십시오.

#### 6. 액세스 경로 추가

버킷 리스트에 대한 액세스 권한이 없는 서브 계정으로 로그인하는 경우, **액세스 경로 추가** 방식을 통해 액세스할 수 있으며, COSBrowser에서 액세스 경로를 추가하는 방법은 다음 두 가지가 있습니다.
(1) 로그인 인터페이스에서 직접 액세스 경로를 추가하고, 해당하는 버킷 리전 정보를 선택한 후 로그인하면 리소스를 관리할 수 있습니다.
<img src="https://main.qcloudimg.com/raw/5fcb01823ae8220e035e86bfd12f047c.png" width="90%">
(2) 서브 계정으로 로그인한 후, 버킷 리스트 페이지의 왼쪽 상단에 있는 [경로 추가]를 클릭하고 지정 경로를 입력해 버킷 관리 리소스로 이동합니다.

#### 10. 파일 동기화

사용자는 파일 동기화 기능을 통해 특정 로컬 폴더에 있는 파일을 버킷에 자동으로 실시간 업로드할 수 있습니다. 자세한 방법은 다음과 같습니다.

(1) 인터페이스 오른쪽 상단에 있는 [동기화]를 클릭합니다.
(2) 팝업 창에서 로컬 폴더와 버킷 디렉터리를 지정합니다.
(3) [동기화 시작]을 클릭하면 파일 동기화 기능이 활성화됩니다.
(4) 동기화 로그에서 파일의 이전 동기화 로그를 확인할 수 있습니다.
<img src="https://main.qcloudimg.com/raw/283c7f9ee254b08561084ece22e2ada2.png" width="90%" />

> !
> - 동기화란 파일 업로드 시, 시스템에서 자동으로 버킷에 동일한 파일이 존재하는지 확인한 후 동기화 기능을 통해 버킷에 존재하지 않는 파일만 업로드하는 기능입니다.
> - 현재는 로컬 파일을 버킷에 동기화 업로드만 지원하며, 역방향은 지원하지 않습니다.
> - 파일 동기화 기능에서는 수동 동기화와 자동 동기화를 제공합니다.

#### 11. 파일 복사 및 붙여넣기

특정 버킷 또는 경로에서 복사할 파일/폴더를 선택하고 인터페이스 상단에 있는 [더보기]에서 [복사]를 클릭하거나 마우스 오른쪽 버튼을 클릭해 [복사]를 클릭하면 파일/폴더가 복사됩니다. 복사 후 **다른 버킷 또는 경로**에 붙여넣을 수 있으며, 일괄 복사 및 붙여넣기를 지원합니다.

> !붙여넣을 대상 경로에 복사한 파일 또는 폴더와 동일한 이름의 파일이 존재하는 경우 기본적으로 덮어쓰기 합니다.

#### 12. 파일 이름 변경

이름을 변경할 파일을 선택하고 마우스 오른쪽 버튼을 클릭하여 [이름 변경]을 클릭하거나 파일 오른쪽에 있는 [더 많은 조작]에서 [이름 변경]을 클릭하여 파일 이름을 입력한 후 확인을 클릭하면 파일의 이름이 변경됩니다.

> ?폴더 이름은 변경할 수 없습니다.

<span id="newfolder"></span>

#### 13. 새 폴더 생성

특정 버킷 또는 경로에서 인터페이스에 있는 [새 폴더 생성]을 클릭하거나 마우스 오른쪽 버튼을 클릭하여 [새 폴더 생성]을 클릭하고 폴더 이름을 입력한 후 확인을 클릭하면 폴더가 생성됩니다.

> !
> - 폴더 이름은 최대 255자까지 입력할 수 있으며, 숫자, 영어 및 출력 가능한 문자를 조합하여 입력할 수 있습니다.
> - 폴더 이름에는 ‘\ / : * ? " | < >’ 등의 특수 문자는 포함될 수 없습니다.
> - ‘..’을 폴더 이름으로 사용할 수 없습니다.
> - 폴더 이름은 변경할 수 없으므로 신중하게 생성하십시오.

#### 15. 파일 링크 생성

COS에 저장된 모든 파일은 관련 링크를 통해 액세스할 수 있습니다. 파일이 개인 읽기 권한이라면 임시 서명을 요청하여 유효 시간이 부여된 임시 액세스 링크를 생성할 수 있습니다.

파일의 링크 생성 방법에는 다음과 같은 방법이 있습니다.

- 리스트에서 파일 오른쪽에 있는 공유 아이콘을 클릭하면 링크 생성과 복사가 한번에 이루어집니다. 파일을 공개 읽기 권한으로 설정할 경우 링크는 서명 없이 영구적으로 유효하게 되며, 개인 읽기 권한으로 설정할 경우 링크에 서명이 존재하여 2시간 내에만 유효하게 생성됩니다.
- 파일 선택 후 오른쪽에 있는 [링크 복사]를 클릭하면 링크 생성과 복사가 한번에 이루어집니다. 파일을 공개 읽기 권한으로 설정할 경우 링크는 서명 없이 영구적으로 유효하게 되며, 개인 읽기 권한으로 설정할 경우 링크에 서명이 존재하여 2시간 내에만 유효하게 생성됩니다.
- 파일 상세 정보에서 [임시 링크 생성]을 클릭하면 도메인을 지정한(예: 활성화한 CDN 가속 도메인 설정 가능) 임시 링크, 링크 유형, 유효 시간을 설정할 수 있습니다.

#### 16. 파일/폴더 공유

작업 열의 [공유] 아이콘 또는 메뉴의 [공유] 옵션을 클릭하여 COS 폴더를 공유할 수 있으며, 공유 유효시간을 설정할 수 있습니다.

>?
>- 단일 폴더 공유만 지원되며, 파일 일괄 공유는 지원되지 않습니다.
>- 여러 사람과 파일을 공유하면 파일 버전이 쉽게 혼동될 수 있습니다. 버킷 버전 제어 기능을 활성화하면 파일의 이전 버전으로 되돌아갈 수 있습니다.

| --- | --- |

| \- 읽기 전용 | 링크에 액세스하여 폴더 리스트 불러오기, 폴더에 있는 파일 다운로드 작업이 가능합니다.|

#### 17. 파일 미리보기

COSBrowser는 미디어 유형 파일에 대한 미리보기를 지원합니다. 현재 이미지, 비디오, 오디오 파일의 미리보기를 지원하며, 미디어 형식 파일 이름을 더블 클릭하거나 마우스 오른쪽 버튼을 클릭하여 메뉴에서 [미리보기] 또는 [재생]을 클릭하면 파일 미리보기 인터페이스가 열립니다. 파일 미리보기 또는 재생 인터페이스에서 다음을 선택할 수 있습니다.

- **링크 복사**: 파일 액세스 링크를 생성 및 복사합니다.
- **다운로드**: 파일을 로컬로 다운로드. 로컬에 동일한 이름의 파일이 존재하는 경우 기본적으로 덮어쓰기 합니다.
- **모바일 보기**: 미리보기 인터페이스에서 파일에 액세스할 수 있는 QR코드를 생성할 수 있으며, 모바일로 스캔하여 모바일 상에서 직접 해당 파일을 볼 수 있습니다.

> !
> - 대다수의 이미지 형식의 미리보기를 지원하며, 비디오는 mp4와 webm 형식만 지원하고 오디오는 mp3, wav 형식만 지원합니다.
> - 파일 미리보기는 다운스트림 트래픽이 발생하므로 유의하여 사용하십시오.
