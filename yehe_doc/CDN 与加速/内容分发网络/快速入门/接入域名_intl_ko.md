서비스 활성화 후 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인해 메뉴바에서 [Domain Management]를 선택하여 가속 도메인을 추가하면 Tencent Cloud에서 제공하는 가속 서비스를 이용할 수 있습니다. 도메인 추가 후에는 관련 도메인 설정이 전체 네트워크 CDN 가속 노드로 전달되며, 현재 네트워크 서비스에 직접적인 영향을 미치지 않습니다. 정식으로 가속을 활성화해야 하는 경우 CNAME 구성이 필요하며, 자세한 절차는 [CNAME 구성](https://intl.cloud.tencent.com/document/product/228/3121)을 참조하십시오.

## 도메인 추가
Domain Management 페이지에서 [Create a Distribution]을 클릭합니다.
![](https://main.qcloudimg.com/raw/02784c06077ee97314ec146aa3369fb1.png)

Create a Distribution 페이지는 세 부분으로 구성되어 있습니다.

- 도메인 속성 설정
- 원본 서버 설정
- 가속 서비스 설정

Tencent Cloud CDN은 선택한 도메인 속성 설정과 원본 서버 설정에 따라 캐시 특성 설정을 추천하며, 수정할 필요 없이 직접 제출할 수 있습니다.

### 1. 도메인 속성 설정
도메인 설정 모듈에 비즈니스 서비스 도메인을 입력하고 해당 프로젝트, 가속 지역, 서비스 유형을 선택합니다.
<img src="https://main.qcloudimg.com/raw/4f2de8b5158a317b1913bd5f920dbf3b.png" style="height:280px"/>

**설정 페이지 상세 설명:**

| 설정 페이지   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 도메인     | 1. 도메인명 길이는 50자를 초과할 수 없습니다.<br/>2. 도메인이 공업정보부의 ICP비안을 받은 상태여야 합니다.<br/>3.도메인이 a.test.com, a.b.test.com 등과 같이 서브도메인 형태이거나 \*.test.com、\*.a.test.com과 같이 와일드카드 서브도메인이어야 합니다.<br/>4. 도메인이 와일드카드 서브도메인이거나 이미 다른 사용자가 액세스한 경우 [모든 권한 인증](#m1) 후 액세스 또는 검색할 수 있습니다.<br/><br/><strong>주의사항:</strong><br/>1. 현재 와일드카드 서브도메인 액세스 후에는 다른 계정으로 서브도메인 또는 2레벨 와일드카드 서브도메인에 액세스할 수 없습니다. <br/>2. 현재 `*.test.com`과 `*.a.test.com`의 동시 액세스를 지원하지 않습니다. |
| 서브 프로젝트 | 프로젝트는 Tencent Cloud의 모든 클라우드 서비스가 공유하는 리소스 그룹의 개념으로, [Project Management](https://console.cloud.tencent.com/project)에서 프로젝트 관련 작업을 할 수 있습니다. |
| 가속 지역 | 중국 내: 글로벌 사용자 액세스가 모두 중국 내 가속 노드로 스케쥴링되어 서비스됩니다.<br/>중국 외: 글로벌 사용자 액세스가 모두 중국 외 가속 노드로 스케쥴링되어 서비스됩니다.<br/>글로벌: 글로벌 사용자 액세스가 가장 근접한 노드로 최적화 스케쥴링되어 서비스됩니다.<br/><br/><strong>주의사항:</strong><br/>중국 내와 중국 외 가속 서비스는 분리 과금되며, 과금 정책은 [조회](https://intl.cloud.tencent.com/document/product/228/2949)를 클릭하여 확인하십시오. |
| 서비스 유형 | Tencent Cloud CDN은 각 서비스 유형별로 타겟팅 가속 성능 최적화를 제공하며,<br/>비즈니스에 더 적합한 서비스 유형을 선택하여 더 우수한 품질의 가속 효과를 누릴 수 있습니다.<br/><br/>정적 가속: 이커머스, 웹 사이트, 게임 이미지 유형의 소형 리소스 가속 시나리오에 적합합니다.<br/>다운로드 가속: 게임 설치 패키지, 멀티미디어 리소스 파일 다운로드, 모바일 펌웨어 배포 등 다운로드 시나리오에 적합합니다.<br/>스트림 미디어 VOD 가속: 온라인 교육, 온라인 비디오 VOD 등의 시나리오에 적합합니다. |
| IPv6 액세스<br>(구: 프로토콜 요청)| IPv6 액세스는 기본적으로 비활성화되어 있습니다. 활성화 후에는 IPv6 프로토콜을 통해 CDN 노드에 액세스하며, 도메인을 추가한 후에는 자유롭게 활성화/비활성화할 수 있습니다.<br/><strong><br/>주의사항: </strong><br/>• IPv6 액세스는 중국 내에서만 지원합니다. 도메인 가속 지역이 글로벌인 상태에서 IPv6 액세스를 활성화하면 중국 내에만 적용되며, 도메인 가속 지역이 중국 외인 경우에는 활성화할 수 없습니다.<br>• 서비스 유형이 “스트림 미디어 VOD 가속”인 경우, 현재는 IPv6 액세스를 지원하지 않습니다. <br>• 일부 플랫폼은 현재 업데이트 중으로 현재 IPv6 액세스를 지원하지 않습니다. 도메인에 특수 설정이 존재하는 경우에도 지원하지 않습니다. | 
| 태그 | 최대 50개의 태그를 추가할 수 있습니다.<br/>인벤토리 태그(태그 콘솔에서 추가한 태그) 선택만 지원합니다.<br/>태그 추가에서 태그 버튼과 태그값은 필수 선택 사항입니다. [태그 관리](https://console.cloud.tencent.com/tag/taglist)에서 태그 관련 작업을 할 수 있습니다. |

### 2. 원본 서버 설정

비즈니스 원본 서버 관련 정보를 설정하며, 캐시에 리소스가 없을 때 CDN 노드가 원본 서버에서 불러와 캐싱합니다.
![](https://main.qcloudimg.com/raw/4a051cfc61dbd0e64b5ccfb8b386829d.png)

**설정 페이지 상세 설명:**

| 설정 페이지   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 원본 서버 유형 | 외부 원본: 이미 안정적으로 운영하고 있는 비즈니스 서버(즉, 원본 서버)가 있는 시나리오<br/><a href = "https://intl.cloud.tencent.com/product/cos">COS(COS 원본)</a>: 리소스가 이미 Tencent Cloud COS에 저장되어 있고 원본 서버로 직접 bucket을 선택할 수 있는 경우 |
| 원본 서버 주소 | 외부 원본:<br/>1. 여러 개의 IP를 원본 서버로 설정할 수 있으며, 원본 가져오기 시 폴링합니다.<br/>2. 포트(0 - 65535)와 가중치(1 - 100) 추가 설정 지원: `원본 서버:포트:가중치`(포트 생략 가능: `원본 서버::가중치`, HTTPS 프로토콜은 현재 443 포트만 지원합니다.<br/>3. 도메인을 원본 서버로 설정할 수 있으며, 해당 도메인은 반드시 서비스 가속 도메인과 달라야 합니다. |
| 원본 가져오기 프로토콜 | 원본 서버 지원 상황에 따라 원본 요청 프로토콜 선택:<br/>HTTP: HTTP/HTTPS 액세스는 모두 HTTP 원본 가져오기를 사용합니다.<br/>HTTPS: HTTP/HTTPS 액세스는 모두 HTTPS 원본 가져오기(원본 서버가 HTTPS 액세스를 지원해야 함)를 사용합니다.<br/> 프로토콜 준수: HTTP 액세스는 HTTP 원본 가져오기를 사용하고, HTTPS 액세스는 HTTPS 원본 가져오기(원본 서버가 HTTPS 액세스를 지원해야 함)를 사용합니다. |
| 원본 가져오기 도메인 | 원본 가져오기 도메인이란 CDN 노드가 원본을 가져올 때 원본 서버에 액세스되는 사이트 도메인입니다.<br/>서브도메인 액세스 시, 기본적으로 가속 도메인과 동일하며 직접 수정할 수 있습니다.<br/>와일드카드 서브도메인 액세스 시, 기본적으로 실제 서브 도메인으로 액세스되며 직접 수정할 수 있습니다. |

### 3. 가속 서비스 설정
노드의 가속 서비스 관련 구성을 설정합니다.
![](https://main.qcloudimg.com/raw/fc616c71acf52fe77af7ce08d9773ad4.png)

**설정 페이지 상세 설명:**

| 설정 페이지   | 설정 설명                                                     |
| :------- | :----------------------------------------------------------- |
| 기본 설정 | 노드 캐시 리소스는 Key-Value에 따라 매핑하며, 여기서 Key가 리소스 URL입니다. 필터링 매개변수를 활성화하는 경우 Key는 URL에서 “?”를 생략한 매개변수를 매핑하며, 필터링 매개변수를 비활성화하는 경우 Key는 전체 리소스 URL이 됩니다. 정적 가속 유형은 기본적으로 비활성화되어 있으며 다운로드, 스트림 미디어 VOD 가속 유형은 기본적으로 활성화되어 있습니다. |
| 샤드 원본 불러오기 | 원본 불러오기 설정 시 샤드를 진행할지 여부를 선택하며, 원본 서버에서 샤드를 지원해야만 활성화할 수 있습니다. COS 원본 서버 또는 서비스 유형이 스트림 미디어 VOD 가속인 경우 기본적으로 샤드 원본 불러오기가 활성화되어 있습니다. |
| 캐시 규칙 | 노드 캐시 만료 시간을 설정합니다. 기본적인 파일 캐시 만료 시간은 정적 가속 유형의 경우 모두 원본 서버를 따르며, 다운로드, 스트림 미디어 VOD 가속 유형의 경우 모두 30일입니다. 노드 캐시 만료 시간을 최대로 설정하는 경우 노드 보관 리소스에 영향을 주며 실제 캐시 시간은 상황에 따라 다릅니다. |

## 액세스 완료

[제출 확인]을 클릭하여 도메인 추가 작업을 완료합니다. 도메인 설정이 전체 네트워크 노드에 전달될 때까지 잠시 기다려 주십시오. 전달 시간은 약 5~10분 소요됩니다.
![](https://main.qcloudimg.com/raw/0eb54022c36df808aabe5fb4c2838c77.png)

## 후속 작업
액세스가 완료되면 Tencent Cloud CDN에서 해당 CNAME 주소를 할당하므로 CDN 서비스를 적용하려면 CNAME 구성을 완료해야 합니다. 자세한 내용은 [CNAME 구성](https://intl.cloud.tencent.com/document/product/228/3121)을 참조하십시오.

## 도메인 소유권 인증
액세스하는 도메인이 와일드카드 서브도메인이거나 기타 사용자가 이미 액세스한 경우, 도메인 소유권 인증 후에만 액세스 또는 검색할 수 있습니다. 도메인 소유권 인증은 DNS 인증 방식이며, 인증 절차는 다음과 같습니다.

1. [인증 방법]을 클릭하여 DNS 인증 시 추가해야 하는 리졸브 기록 정보를 획득합니다. 인증 완료 전에는 해당 페이지를 닫지 마십시오.
   ![img](https://main.qcloudimg.com/raw/07b8c6621e0f8d3588373a6889477a6b.png)
2. 리졸브에 TXT 유형의 DNS 기록을 추가하고 CVM 기록에 _cdnauth를 입력하여, 인증 방법에서 랜덤으로 기록값을 생성할 수 있도록 기록값을 설정합니다.
3. TXT 리졸브가 적용될 때까지 기다린 후 [인증] 버튼을 클릭하여 인증을 진행합니다. “인증 실패”가 표시되는 경우 DNS 기록이 적용될 때까지 기다린 후 다시 시도해 주시고, TXT 기록이 정상적으로 작성되는지 확인합니다.
