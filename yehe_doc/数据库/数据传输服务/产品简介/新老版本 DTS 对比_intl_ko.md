## 개요
DTS의 레거시 버전은 클라우드 네이티브 복제를 통해 증분 데이터 동기화를 구현하여, 원본 데이터베이스의 커널 형식에 대한 의존도가 높고 기능에 많은 제한이 있습니다. 예를 들어 테이블 이름 매핑 및 다대일 데이터베이스 동기화와 같은 고급 기능을 지원하지 않습니다.

신규 버전 DTS는 새로운 독점 커널을 사용하고 제품 기능을 포괄적으로 개선하였습니다. 안정성, 가용성, 유연성, 안전성이 높은 데이터 전송 기능을 제공하여 사용 편의성을 높였습니다. 이종 데이터베이스 마이그레이션 및 교차 계정 데이터 마이그레이션과 같은 고급 기능을 통해 데이터 전송 기능이 개선되었습니다. 

DTS의 신규 버전은 점차적으로 레거시 버전을 대체할 예정이며, 레거시 버전은 2021년 말에 사용 중지되므로 지원한 한 빨리 신규 버전으로 업그레이드하시기 바랍니다. 

## 기능 비교
### 데이터 마이그레이션

| -------------- | --------------------------------- | --------------------------------- |
| 데이터베이스 유형     | <li>이종 데이터베이스 마이그레이션 미지원.<br><li>Alibaba Cloud 원본 데이터베이스 마이그레이션 지원. | <li>이종 데이터베이스 마이그레이션 지원.<br><li>Alibaba Cloud 및 AWS 원본 데이터베이스 마이그레이션 지원. |
| 교차 계정 마이그레이션     | 교차 계정 마이그레이션 미지원.          | 교차 계정 마이그레이션 지원.                                             |
| 지원되는 버전     | 버전 간 마이그레이션 미지원.         | 대다수의 데이터베이스 버전 간 마이그레이션 지원.  |

| 향상된 운영 경험   | -                                     | <li>기본 키 또는 null이 아닌 고유 키가 있는 테이블의 경우 잠금 없는 마이그레이션을 지원합니다.<br><li>테이블 매핑 지원.<br><li>증분 마이그레이션 중 인스턴스 재시작 또는 업그레이드 지원.<br><li>데이터베이스에서 마이그레이션 작업을 분리하고 증분 마이그레이션 중에 잠금을 해제합니다. |

### 데이터 구독
DTS의 신규 버전과 레거시 버전의 주요 기능 비교는 다음과 같습니다.

| **비교 항목**       | **레거시 버전**                | **신규 버전**                                           |
| ---------------- | --------------------------------- | -------------------------------------------------- |
| 데이터베이스 유형       | TencentDB for MySQL만 지원. | TencentDB for MySQL, TencentDB for MariaDB, 및 TDSQL for MySQL 모두 지원. |
| 구독 데이터 저장 방법 | 데이터는 프라이머리/세컨더리 모드의 물리적 시스템에 저장됩니다.    | 데이터는 Kafka 미들웨어에 저장되며, 단일 Topic에 대한 데이터의 분할 저장을 지원(파티셔닝 정책 지원)하여 소비 효율성을 높입니다. |
| 지원되는 프로토콜       | SDK for Java 전용 프로토콜.       | 여러 프로그래밍 언어용 Kafka Client SDK가 있는 Kafka 프로토콜.           |
| 모니터링 및 알람 | <li>메트릭 모니터링 지원.<li>기본 알람 정책 미지원. | <li>더 많은 메트릭 모니터링 지원.<li>기본 알람 정책 지원, 구독 중 예외 알람 자동 트리거. |

| 소비 방법         | 직렬 처리 지원.                  | 여러 파티션에서 데이터를 동시에 소비하기 위해 단일 Topic에서 데이터의 분할된 저장을 지원하여 소비 효율성을 향상시킵니다. |
