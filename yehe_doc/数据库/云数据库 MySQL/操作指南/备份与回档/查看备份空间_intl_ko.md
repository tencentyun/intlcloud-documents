본 문서는 콘솔에서 MySQL 인스턴스 백업 용량 및 무료 한도를 조회하는 방법에 대해 소개합니다.

## 작업 시나리오
TencentDB for MySQL 인스턴스 백업 파일에 의해 점유되는 백업 용량은 리전에 따라 할당되며, 자동 데이터 백업, 수동 데이터 백업 및 로그 백업을 포함하여 해당 리전에 있는 모든 MySQL 데이터베이스 백업의 스토리지 용량 합계를 말합니다. 백업 보관 시간을 연장하거나 수동 백업 빈도를 늘리면 데이터베이스 백업의 스토리지 용량이 늘어나게 됩니다.

## 작업 순서
1. [TencentDB for MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 후, 왼쪽 메뉴에서 [데이터베이스 백업] 페이지를 선택합니다.
2. 상단에서 리전을 선택한 다음, '개요' 페이지에서 해당 리전의 백업 총계, 추세, 통계 등 백업 정보를 확인할 수 있습니다.
 - 백업 총계: 총 백업, 데이터 백업, 로그 백업의 용량 크기 및 수량을 나타내며, 총 백업의 무료 용량 점유율을 보여줍니다. 
>?
> - 초록색: 사용한 총 백업 양이 무료 용량을 초과하지 않았음을 의미합니다.
> - 노란색: 사용한 총 백업 양이 무료 용량을 초과해 요금이 발생했음을 의미합니다. 자세한 내용은 [백업 용량 요금 설명](https://intl.cloud.tencent.com/document/product/236/32344)을 참조하십시오.
>
![](https://main.qcloudimg.com/raw/e9489e74614d7708d357de6943837c3c.png)
 - 백업 추세: 각 총계 데이터의 추세를 나타냅니다.
 - 백업 통계: 해당 리전에 있는 인스턴스의 인스턴스 ID/이름(클릭하면 인스턴스 상세 페이지로 이동), 백업 용량(용량에 따른 정렬 지원), 데이터 및 로그 백업 등의 정보를 보여줍니다. 오른쪽 상단 검색창에서 인스턴스 이름/ID를 입력해 검색할 수 있습니다.
3. 상단에서 [백업 리스트] 페이지를 선택합니다. 백업 리스트는 데이터 백업 리스트와 로그 백업 리스트로 구분됩니다. 리스트에서 인스턴스 이름을 클릭하면 인스턴스 상세 페이지로 이동합니다. 백업 리스트는 시간별 필터 및 인스턴스 이름/ID 퍼지 검색을 지원합니다.
![](https://main.qcloudimg.com/raw/0587e4f22b33960c349b2e2dbbe81d10.png)
 - **데이터 백업 리스트**
    - 정보 필드 필터 지원:
유형: 전체, 로직 콜드 스탠바이, 물리 콜드 스탠바이
백업 방식: 전체, 자동, 수동
백업 방법: 현재 전체 백업만 지원합니다.
    - 백업 시점, 작업 시작/종료 시간, 백업 크기에 따른 정렬을 지원합니다.
    - 작업 열에서 [상세 내용]을 클릭하면 인스턴스 백업 복구 페이지로 이동할 수 있으며, [다운로드]를 클릭하면 백업을 다운로드할 수 있습니다. [삭제] 작업은 수동 백업에서만 지원합니다.
 - **로그 백업 리스트**
    - 로그 데이터의 시작/종료 시간에 따른 필드 정렬을 지원합니다.
    - 작업 열에서 [상세 내용]을 클릭하면 인스턴스 백업 복구 페이지로 이동할 수 있으며, [다운로드]를 클릭하면 로그를 다운로드할 수 있습니다.
